# Phase 6.4 — Discovery Mode Design Spec

**Date:** 2026-03-22
**Status:** Approved
**Effort:** M (half day)

---

## Overview

Phase 6.4 builds ShellQL's connection discovery layer — a persistent, aggregated store of
database connections sorted by last-used. It replaces the flat recent-files list with a
proper SQLite metadata database, adds a multi-driver connection record format, and lays
the groundwork for transparent sigil credential integration.

---

## Goals

- Track every opened database connection with enough detail to quickly identify it
- Sort connections by last-accessed so the most relevant appear first
- Aggregate ShellQL's own store with sigil's database entries into a single list
- Support the full connection record shape for future MySQL/PostgreSQL drivers
- Maintain a clean seam for sigil credential lookup (opt-in, not automatic)

---

## Non-Goals (deferred)

- Network driver support (MySQL, PostgreSQL, MSSQL) — record format supports it; wiring deferred
- UI/UX polish — scheduled after Phase 6.5 (v0.1.0 alpha milestone)
- sigil auto-registration — connections are not written to sigil automatically
- Sanctum daemon / IPC — sigil is invoked as a CLI subprocess only
- `[o] Open new` welcome screen key — remains a no-op (TODO); needs file-picker primitive

## Architecture

### New files

- `src/connections.sh` — connection store: owns `shellql.db`, all CRUD, last_accessed
  tracking, aggregate list, migration

### Changed files

- `src/state.sh` — remove `shql_state_load_recent`, `shql_state_push_recent`,
  `SHQL_HISTORY_FILE`, and `SHQL_RECENT_MAX`; add four new parallel arrays
- `bin/shql` — source `connections.sh`; call `shql_conn_init` + `shql_conn_migrate` at

  `shql_conn_push`; update `databases` dispatch
- `src/screens/welcome.sh` — five specific change sites (see Integration section)
- `src/db_mock.sh` — extend `shql_mock_load_recent` to also populate the four
  `SHQL_RECENT_*` parallel arrays (see Mock Mode section)

### Data store

`$SHQL_DATA_DIR/shellql.db` — ShellQL's own SQLite metadata database.
`SHQL_DATA_DIR` follows the XDG Base Directory spec:
```bash
SHQL_DATA_DIR="${XDG_DATA_HOME:-$HOME/.local/share}/shellql"
```
Default location: `~/.local/share/shellql/shellql.db`. No user setup required.

## Data Model

```sql
CREATE TABLE IF NOT EXISTS connections (
    id         TEXT PRIMARY KEY,
    driver     TEXT NOT NULL,      -- 'sqlite' | 'mysql' | 'postgresql' | 'mssql'
    name       TEXT NOT NULL,      -- display name (see Name Derivation below)
    path       TEXT NOT NULL DEFAULT '',
    host       TEXT NOT NULL DEFAULT '',
    port       TEXT NOT NULL DEFAULT '',
    user       TEXT NOT NULL DEFAULT '',
    db_name    TEXT NOT NULL DEFAULT '',
    sigil_name TEXT NOT NULL DEFAULT ''   -- optional; links to a sigil entry name
);

-- Unique constraints are partial to avoid false conflicts on the empty-string default.
-- path uniqueness covers SQLite connections; host+port+db_name covers network drivers.
CREATE UNIQUE INDEX IF NOT EXISTS uq_connections_path
    ON connections (path) WHERE path != '';
CREATE UNIQUE INDEX IF NOT EXISTS uq_connections_network
    ON connections (host, port, db_name) WHERE host != '';

CREATE TABLE IF NOT EXISTS last_accessed (
    source    TEXT NOT NULL,       -- 'local' | 'sigil'
    ref_id    TEXT NOT NULL,       -- connections.id  OR  sigil entry name (@env suffix if scoped)
    last_used TEXT NOT NULL,       -- ISO8601 UTC e.g. '2026-03-22T10:00:00Z'
    PRIMARY KEY (source, ref_id)
);

CREATE INDEX IF NOT EXISTS idx_last_accessed_ref ON last_accessed (ref_id);
```

Note: SQLite foreign key enforcement requires `PRAGMA foreign_keys = ON`. ShellQL's
metadata DB does not enable this pragma — orphan `last_accessed` rows (if a connection
row is ever deleted) are acceptable. The index on `ref_id` ensures the JOIN in
`shql_conn_list` does not degrade to a full scan.

### Name derivation

- **SQLite:** last two path segments — `/Users/allen/projects/myapp/db.sqlite` →
  `myapp/db.sqlite`
- **Network drivers:** `db_name` alone — e.g. `myapp`

### UUID generation

`uuidgen` preferred (available on macOS and most Linux). Fallbacks in order:
1. `cat /proc/sys/kernel/random/uuid` (Linux without uuidgen)
2. `printf '%s-%s' "$(date +%s)" "$$"` (seconds + PID, portable macOS/Linux)

Note: `date +%s%N` (nanoseconds) is Linux-only — BSD `date` on macOS prints `%N`
literally. The third fallback is used only on unusual Linux systems without `uuidgen`
or `/proc`.

### Upsert semantics

`shql_conn_push` must preserve the existing `id` on update (not regenerate it), because
`last_accessed.ref_id` references it. The implementation must:
1. `SELECT id FROM connections WHERE (path=? AND path != '') OR (host=? AND host != '' AND port=? AND db_name=?)`
2. If found: `UPDATE connections SET ... WHERE id=<existing_id>`
3. If not found: generate UUID, `INSERT INTO connections ...`

`INSERT OR REPLACE` is explicitly forbidden here — it generates a new `id`, orphaning
the `last_accessed` row.

### Connection types

**Local (discrete):** stored in ShellQL's `connections` table. May optionally set
`sigil_name` to link to a sigil entry (e.g. for future password lookup).

**Sigil-sourced:** discovered at runtime via `sigil list --type database --porcelain`.
Not stored in `connections`; tracked only in `last_accessed` with `source='sigil'`.

### Deduplication

A local connection with `sigil_name` set and a sigil entry with that name are the same
logical connection. The local record is authoritative; the sigil entry does not create a
second row in the aggregate list.

## `src/connections.sh` — Function Contracts

```
shql_conn_init
  Ensure shellql.db exists with current schema. Creates SHQL_DATA_DIR if needed.
  No-op if already initialised. Errors to stderr; returns 1 if sqlite3 unavailable
  or directory cannot be created.
  Not called in mock mode (see bin/shql changes).

shql_conn_push <driver> <path> [host] [port] [user] [db_name] [sigil_name]
  Upsert connection row using the preserve-id upsert semantics above.
  Then upsert last_accessed (source='local', ref_id=<connections.id>) with current
  UTC timestamp.
  Name is auto-derived internally (see Name Derivation above) — not a parameter.
  Fully silent on failure: exit 0, no stdout, no stderr. A failed history write must
  never abort or produce noise in the caller's output.

shql_conn_list
  Print aggregate connection list to stdout as data rows only (no header line),
  sorted by last_used DESC. Format: tab-delimited, one line per connection:
    <source>\t<driver>\t<name>\t<detail>\t<last_used>
  source  = 'local' | 'sigil'
  detail  = path (sqlite) | host:port/db_name (network)
  last_used = ISO8601 string, or empty string for connections never accessed.

  Sort query uses COALESCE to place never-accessed connections last:
    ORDER BY COALESCE(la.last_used, '') DESC
  An empty string sorts before all ISO8601 timestamps under text comparison,
  so never-accessed entries always appear at the bottom.

  Calls `sigil list --type database --porcelain` if sigil is on PATH.
  Silently skips sigil aggregation if sigil is absent or exits non-zero.

shql_conn_load_recent
  Populate four parallel arrays from shql_conn_list output:
    SHQL_RECENT_NAMES[]   — display strings for the list widget (e.g. 'myapp/db.sqlite')
    SHQL_RECENT_DETAILS[] — detail column value only (path for SQLite; host:port/db_name
                            for network). Does NOT include the driver prefix.
    SHQL_RECENT_SOURCES[] — 'local' | 'sigil'
    SHQL_RECENT_REFS[]    — connections.id (local) | sigil entry name (sigil)
  Replaces shql_state_load_recent.

  Implementation note: to populate global arrays, capture shql_conn_list output
  into a variable first, then iterate with `while IFS=$'\t' read -r ...` over the
  variable (not a subshell pipeline). This is required for bash 3.2 compatibility —
  assignments inside a subshell (e.g. `while ... done < <(...)`) do not persist to
  the calling environment. See shql_state_load_recent in state.sh (line 38) for the
  established pattern.

shql_conn_migrate
  One-time migration from legacy flat file ($SHQL_DATA_DIR/recent) to shellql.db.
  Imports each non-empty path line as a local SQLite connection.

  Must insert into `connections` directly (NOT via shql_conn_push) to avoid
  creating last_accessed entries. Migrated connections must appear as never-accessed
  in the aggregate list.

  All-or-nothing: wrap all inserts in a single SQLite transaction. On any failure,
  ROLLBACK and leave `recent` intact. Only COMMIT and rename recent → recent.bak
  after all rows insert successfully.

  Silent no-op if recent file does not exist (normal first-run case).
```

## `bin/shql` Changes

**Source order:** add `source "$_SHQL_ROOT/src/connections.sh"` after `state.sh`.

**Startup:** call init and migrate only in real mode (mock mode has no metadata DB):
```bash
if ! (( SHQL_MOCK )); then
    shql_conn_init || exit $?
    shql_conn_migrate
fi
```

**Push on open:** in `open`, `table`, `query-tui`, `query-out`, `pipe` cases — replace
`shql_state_push_recent "$SHQL_DB_PATH"` with `shql_conn_push "sqlite" "$SHQL_DB_PATH"`.
In mock mode, `shql_conn_push` is a no-op (mock has no DB); no guard needed since the
function exits silently on failure.

**`databases` dispatch:** replace the existing loop with:
```bash
databases)
    _list=$(shql_conn_list)
    if (( ${_SHQL_CLI_PORCELAIN:-0} )); then
        printf '%s\n' "$_list"
    else
        _header=$'source\tdriver\tname\tdetail\tlast_used'
        shql_cli_format_table "${_header}"$'\n'"${_list}"
    fi
    ;;
```
The header line is prepended in the dispatch block, not emitted by `shql_conn_list`,
so other callers of `shql_conn_list` receive clean data rows.

## Error Handling

| Scenario | Behaviour |
|---|---|
| `sqlite3` not on PATH (real mode) | `shql_conn_init` prints error to stderr, returns 1 |
| `$SHQL_DATA_DIR` not creatable | `shql_conn_init` prints error to stderr, returns 1 |
| `shql_conn_push` fails for any reason | Fully silent — exit 0, no stdout, no stderr |
| `shellql.db` unreadable in `shql_conn_list` | Print available results; degrade gracefully |
| `sigil` not on PATH | Skip sigil aggregation silently |
| `sigil list` exits non-zero | Skip sigil aggregation silently |
| Migration fails partway | ROLLBACK transaction; leave `recent` intact |
| No `recent` file on first run | Silent no-op |

## Cross-Repo Dependency

**sigil — `[CLI] Add --porcelain flag to sigil list`**
`shql_conn_list` needs `sigil list --type database --porcelain` for machine-readable output.
Until this lands, the sigil aggregation path in `shql_conn_list` is a no-op (sigil on PATH
but `--porcelain` not available → `sigil list` exits non-zero → skip silently).
Ticket to be created in `fissible/sigil-workspace`.

**sigil — `sigil get <name> <field>` interface**
Used in `welcome.sh` change site 3 to resolve a sigil-sourced connection's path.
Interface: `sigil get <entry-name> <field>` prints the field value to stdout, exit 0 on
success, non-zero if the entry or field is not found. No decoration — suitable for
`$()` capture. This interface is stable as of sigil v0.2.1.

## Migration Path

1. On first run after upgrade, `shql_conn_migrate` reads `$SHQL_DATA_DIR/recent`
   (one path per line) and inserts each as a local SQLite connection directly into the
   `connections` table inside a single transaction — no `last_accessed` rows created,
   so migrated connections sort to the bottom of the list as never-accessed.
2. On COMMIT success: rename `recent` → `recent.bak`.
3. On any failure: ROLLBACK; leave `recent` intact for retry on next startup.
4. Subsequent runs: `recent.bak` ignored; migration is a no-op.
