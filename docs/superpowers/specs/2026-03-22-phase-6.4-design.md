## Overview

Phase 6.4 builds ShellQL's connection discovery layer â€” a persistent, aggregated store of
database connections sorted by last-used. It replaces the flat recent-files list with a
proper SQLite metadata database, adds a multi-driver connection record format, and lays
the groundwork for transparent sigil credential integration.

---

## Goals

- Track every opened database connection with enough detail to quickly identify it
- Sort connections by last-accessed so the most relevant appear first
- Aggregate ShellQL's own store with sigil's database entries into a single list
- Support the full connection record shape for future MySQL/PostgreSQL drivers
- Maintain a clean seam for sigil credential lookup (opt-in, not automatic)

---

## Non-Goals (deferred)

- Network driver support (MySQL, PostgreSQL, MSSQL) â€” record format supports it; wiring deferred
- UI/UX polish â€” scheduled after Phase 6.5 (v0.1.0 alpha milestone)
- sigil auto-registration â€” connections are not written to sigil automatically
- Sanctum daemon / IPC â€” sigil is invoked as a CLI subprocess only
- `[o] Open new` welcome screen key â€” remains a no-op (TODO); needs file-picker primitive

## Architecture

### New files

- `src/connections.sh` â€” connection store: owns `shellql.db`, all CRUD, last_accessed
  tracking, aggregate list, migration

### Changed files

- `src/state.sh` â€” remove `shql_state_load_recent` / `shql_state_push_recent`; add four
  new parallel arrays for the welcome screen list widget
- `bin/shql` â€” source `connections.sh`; call `shql_conn_init` + `shql_conn_migrate` at
  startup; replace `shql_state_push_recent` calls with `shql_conn_push`; update
  `databases` dispatch to call `shql_conn_list`
- `src/screens/welcome.sh` â€” replace `shql_state_load_recent` with `shql_conn_load_recent`;
  populate list from `SHQL_RECENT_NAMES`; update action handler to resolve connection from
  `SHQL_RECENT_REFS` + `SHQL_RECENT_SOURCES`

### Data store

`$SHQL_DATA_DIR/shellql.db` â€” ShellQL's own SQLite metadata database.
`SHQL_DATA_DIR` follows the XDG Base Directory spec:
```bash
SHQL_DATA_DIR="${XDG_DATA_HOME:-$HOME/.local/share}/shellql"
```
Default location: `~/.local/share/shellql/shellql.db`. No user setup required.

## Data Model

```sql
CREATE TABLE IF NOT EXISTS connections (
    id         TEXT PRIMARY KEY,   -- UUID
    driver     TEXT NOT NULL,      -- 'sqlite' | 'mysql' | 'postgresql' | 'mssql'
    name       TEXT NOT NULL,      -- display name; SQLite: last 2 path segments
    path       TEXT NOT NULL DEFAULT '',
    host       TEXT NOT NULL DEFAULT '',
    port       TEXT NOT NULL DEFAULT '',
    user       TEXT NOT NULL DEFAULT '',
    db_name    TEXT NOT NULL DEFAULT '',
    sigil_name TEXT NOT NULL DEFAULT ''  -- optional; links to a sigil entry name
);

CREATE TABLE IF NOT EXISTS last_accessed (
    source    TEXT NOT NULL,       -- 'local' | 'sigil'
    ref_id    TEXT NOT NULL,       -- connections.id  OR  sigil entry name (@env suffix if scoped)
    last_used TEXT NOT NULL,       -- ISO8601 UTC e.g. '2026-03-22T10:00:00Z'
    PRIMARY KEY (source, ref_id)
);
```

### Connection types

**Local (discrete):** stored in ShellQL's `connections` table. May optionally set
`sigil_name` to link to a sigil entry (e.g. for future password lookup).

**Sigil-sourced:** discovered at runtime via `sigil list --type database --porcelain`.
Not stored in `connections`; tracked only in `last_accessed` with `source='sigil'`.

### Deduplication

A local connection with `sigil_name` set and a sigil entry with that name are the same
logical connection. The local record is authoritative; the sigil entry does not create a
second row in the aggregate list.

### SQLite display name

Auto-derived from the last two path segments:
- `/Users/allen/projects/myapp/myapp.sqlite` â†’ `myapp/myapp.sqlite`

### UUID generation

`uuidgen` preferred. Fallbacks in order:
1. `cat /proc/sys/kernel/random/uuid`
2. `date +%s%N` (nanosecond timestamp â€” not a UUID but unique enough)

## `src/connections.sh` â€” Function Contracts

```
shql_conn_init
  Ensure shellql.db exists with current schema. Creates SHQL_DATA_DIR if needed.
  No-op if already initialised. Errors to stderr; returns 1 if sqlite3 unavailable
  or directory cannot be created.

shql_conn_push <driver> <path> [host] [port] [user] [db_name] [sigil_name]
  Upsert connection row (insert if new, update fields if existing path/host matches).
  Upsert last_accessed with current UTC timestamp.
  SQLite: name auto-derives from last two path segments.
  Silent no-op on failure â€” a failed history write must never abort open/query.

shql_conn_list
  Print aggregate connection list to stdout, sorted by last_used DESC.
  Format: tab-delimited, one line per connection:
    <source>\t<driver>\t<name>\t<detail>\t<last_used>
  source  = 'local' | 'sigil'
  detail  = path (sqlite) | host:port/db_name (network)
  Calls `sigil list --type database --porcelain` if sigil is on PATH.
  Silently skips sigil aggregation if sigil is absent or exits non-zero.
  Connections never accessed show last_used as '(never)'.

shql_conn_load_recent
  Populate four parallel arrays from shql_conn_list output:
    SHQL_RECENT_NAMES[]   â€” display strings for the list widget
    SHQL_RECENT_DETAILS[] â€” driver + detail for the status bar
    SHQL_RECENT_SOURCES[] â€” 'local' | 'sigil'
    SHQL_RECENT_REFS[]    â€” connections.id (local) | sigil entry name (sigil)
  Replaces shql_state_load_recent.

shql_conn_migrate
  One-time migration from legacy flat file ($SHQL_DATA_DIR/recent) to shellql.db.
  Imports each path as a local SQLite connection.
  Renames recent â†’ recent.bak only after full successful import.
  Silent no-op if recent file does not exist (normal first-run case).
```

## Error Handling

| Scenario | Behaviour |
|---|---|
| `sqlite3` not on PATH | `shql_conn_init` prints error to stderr, returns 1 |
| `$SHQL_DATA_DIR` not creatable | `shql_conn_init` prints error to stderr, returns 1 |
| `shql_conn_push` fails | Silent no-op â€” history write never aborts a user operation |
| `shellql.db` unreadable in `shql_conn_list` | Print available results; degrade gracefully |
| `sigil` not on PATH | Skip sigil aggregation silently |
| `sigil list` exits non-zero | Skip sigil aggregation silently; log to stderr at debug level |
| Migration fails partway | Leave `recent` intact; do not rename to `.bak` |
| No `recent` file on first run | Silent no-op |

## Migration Path

1. On first run after upgrade, `shql_conn_migrate` reads `$SHQL_DATA_DIR/recent` (one
   path per line) and inserts each as a local SQLite connection with no `last_used` record.
2. After successful import, renames `recent` â†’ `recent.bak`.
3. Subsequent runs: `recent.bak` ignored; migration is a no-op.
